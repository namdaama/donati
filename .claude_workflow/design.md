# 設計: Header ミニマル化対応 (Issue #132)

## 前段階の確認
`.claude_workflow/requirements.md`を読み込みました。

## 現状分析

### 現在のHeader.astro構成
1. **ロゴ**: 左上にhomeLogoがあり、トップページ(/)へのリンク
2. **デスクトップナビゲーション**: 6つのアイコンナビゲーション
   - サイエンス分野 (`/service-fuji`)
   - スペース分野 (`/service-hide`)
   - 活動経歴 (`/professional-experience`)
   - 私たちについて (`/about`)
   - よくある質問 (`/faq`)
   - お問合せ (`/contact`)
3. **モバイルメニュー**: ハンバーガーメニューで同じ6項目を表示
4. **装飾**: 波線（waveLine_752px.svg）

## 実装アプローチ

### 対応方針
要件に基づき、以下の4つのナビゲーションのみを表示：
1. トップページ (`/`) - ロゴがリンクとして機能
2. サイエンス事業 - **既存の「サイエンス分野」(`/service-fuji`)をそのまま使用**
3. 私たちについて (`/about`)
4. お問い合わせ (`/contact`)

### 技術的アプローチ

#### 1. navIconsの絞り込み
`navIcons`配列を以下のように変更：
- **残すもの**:
  - サイエンス分野 (`/service-fuji`) - **そのまま維持**
  - 私たちについて (`/about`) - そのまま維持
  - お問合せ (`/contact`) - そのまま維持
- **コメントアウトするもの**:
  - スペース分野 (`/service-hide`)
  - 活動経歴 (`/professional-experience`)
  - よくある質問 (`/faq`)

#### 2. ロゴの処理
- ロゴはトップページへのリンクなので、そのまま維持

#### 3. 装飾（波線）
- 波線はデザイン要素なので維持

#### 4. モバイルメニュー
- デスクトップと同様に、同じ3項目のみ表示
- 「ホームへ戻る」ボタンは維持（トップページへのリンクとして機能）

## リスクと対策

### リスク1: レスポンシブレイアウトの崩れ
- **問題**: アイコン数が6→3に減ることでレイアウトが不自然になる可能性
- **対策**: 既存のスタイル（`space-x-2 xl:space-x-3`）がそのまま使えるため、問題なし

### リスク2: コメントアウト部分の保守性
- **問題**: 将来的に復元する際、コメント部分を見落とす可能性
- **対策**: コメント部分の先頭に「一時的にコメントアウト (Issue #132)」などの注釈を追加
